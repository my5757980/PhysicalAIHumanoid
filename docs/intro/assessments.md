# Assessments & Grading

This course uses a combination of hands-on labs, projects, and a capstone to assess your learning.

## Assessment Overview

| Component | Weight | Due Date |
|-----------|--------|----------|
| Weekly Labs | 40% | End of each week |
| Midterm Project | 20% | Week 7 |
| Final Capstone | 30% | Week 13 |
| Participation | 10% | Ongoing |

## Weekly Labs (40%)

Each chapter includes hands-on lab exercises that reinforce the concepts covered.

### Lab Structure

- **Duration**: 2-4 hours per lab
- **Submission**: GitHub repository with code and documentation
- **Grading**: Rubric-based (functionality, code quality, documentation)

### Lab Topics by Chapter

| Chapter | Lab Topic |
|---------|-----------|
| 1 | Development environment setup |
| 2 | Kinematic calculations |
| 3 | ROS 2 publisher/subscriber |
| 4 | URDF modeling and TF2 |
| 5 | Gazebo world building |
| 6 | Unity-ROS integration |
| 7 | Isaac Sim basics |
| 8 | Object detection pipeline |
| 9 | Locomotion controller |
| 10 | Speech integration |

### Grading Rubric (per lab)

| Criterion | Points | Description |
|-----------|--------|-------------|
| Functionality | 40 | Code works as specified |
| Code Quality | 25 | Clean, readable, well-organized |
| Documentation | 20 | README, comments, explanations |
| Testing | 15 | Unit tests or demo videos |

## Midterm Project (20%)

**Due: Week 7**

Create a simulated robot that demonstrates mastery of Modules 1-2.

### Requirements

1. **Custom Robot Model**: URDF with at least 6 DoF
2. **Simulated Environment**: Gazebo or Unity world
3. **Autonomous Behavior**: Navigation or simple task execution
4. **Documentation**: Technical report (3-5 pages)

### Deliverables

- [ ] GitHub repository with all code
- [ ] Video demonstration (3-5 minutes)
- [ ] Technical documentation
- [ ] Presentation slides

### Grading Criteria

| Criterion | Points |
|-----------|--------|
| Technical Implementation | 40 |
| Creativity & Design | 20 |
| Documentation Quality | 20 |
| Presentation | 20 |

## Final Capstone (30%)

**Due: Week 13**

Build an autonomous humanoid robot system that demonstrates integration of all course concepts.

### Requirements

1. **Perception**: Computer vision for environment understanding
2. **Planning**: Autonomous decision-making
3. **Control**: Stable locomotion or manipulation
4. **Interaction**: Voice or gesture-based human interaction
5. **Integration**: All components working together

### Suggested Project Types

| Type | Description | Complexity |
|------|-------------|------------|
| Virtual Humanoid | Full humanoid in Isaac Sim | High |
| Physical Assistant | Jetson-based helper robot | Medium-High |
| Manipulation System | Pick-and-place with vision | Medium |
| Navigation Robot | Autonomous explorer | Medium |

### Deliverables

- [ ] GitHub repository with complete codebase
- [ ] Video demonstration (5-10 minutes)
- [ ] Technical report (10-15 pages)
- [ ] Live demo during finals week
- [ ] Presentation (15 minutes + Q&A)

### Grading Criteria

| Criterion | Points |
|-----------|--------|
| Technical Depth | 30 |
| Integration & Completeness | 25 |
| Innovation | 15 |
| Documentation | 15 |
| Presentation & Demo | 15 |

## Participation (10%)

Active engagement in the learning community:

- Discussion forum contributions
- Peer code reviews
- Helping classmates
- Office hours attendance

## Late Policy

| Submission Timing | Penalty |
|-------------------|---------|
| On time | Full credit |
| 1-24 hours late | -10% |
| 1-3 days late | -25% |
| 3-7 days late | -50% |
| >7 days late | No credit |

*Extensions available for documented emergencies*

## Academic Integrity

- All code must be your own work
- External libraries and references must be cited
- Collaboration encouraged for learning, not copying
- AI tools (GitHub Copilot, ChatGPT) allowed with disclosure
